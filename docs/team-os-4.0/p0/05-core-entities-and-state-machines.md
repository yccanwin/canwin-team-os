## 核心实体与最小粒度

| 模块 | 实体 | 最小粒度/唯一事实 |

| 人 | 公司 | 一套部署只初始化一家公司；不建设多租户切换 |
| 人 | 成员岗位 | 一名成员一个主岗位，可有多个附加角色/技能 |
| 人 | 责任路由 | 主管关闭或无有效主管时回落管理员 |
| 场 | 结算客户 | 签约、欠款、付款、退款和开票主体 |
| 场 | 品牌 | 归属于结算客户，聚合多个门店 |
| 场 | 门店 | 交付、异常、续费和案例的业务单位 |
| 场 | 联系人—门店关系 | 多对多；每个联系人最多一个主要门店关系 |
| 场 | 门店商品订阅 | `门店 × 商品订阅`，独立保存开通日、到期日和原订单 |

## 状态与不可绕过条件

### 工作项

`待处理 → 处理中 → 等待他人 → 已完成`，并允许从未完成态进入 `已取消`。

- 提醒/手工任务可以通用完成。
- 收款、出库、验收、冲销等业务动作任务不能通用完成；只能由对应服务器事务成功后关闭。
- 同一业务事件只生成一次工作项，多次点击不得重复记账或重复建单。

### 线索与客户生命周期

`新建 → 跟进/沉淀 → 商机/失败/放弃入公海 → 领取或清理`。

- 未跟进且无关联记录的误建线索，创建人可在 24 小时内删除。
- 已跟进线索只能放弃回区域公海。
- 公海 20 天清理与领取使用同一行锁，领取优先。
- 存在商机、报价、订单、收付款、履约、订阅或案例任一记录时绝不物理删除。
- 竞品老店按到期 60/30/15 天回到工作台；到期未知必须有下次核实时间。

### 报价、订单、库存与履约

`报价草案/改版/审批/过期 → 冻结报价快照 → 正式订单 → 客户款与内部款条件 → 预留/出库 → 分项履约 → 验收/交接`。

- 报价阶段库存绝不变化。
- 正式订单冻结软件、硬件、服务、三种价格、积分、利润比例和劳动收益规则。
- 未结内部款不得启动软件、硬件或服务履约。
- 硬件预留和出库只能针对订单冻结 SKU/数量；不得额外 SKU、超量、负库存或完成后残留预留。
- 取消未出库订单释放预留；已出库只能退库/冲销。

### 财务事件

`待确认 → 已确认` 或 `待确认 → 已驳回`；已确认记录只能追加 `反向冲销/调整`。

- 已确认资金记录不可编辑、不可删除。
- 客户退款先重算订单净收款，再追加利润反向调整。
- 已结算利润不修改原支付记录；不足形成销售债务。
- 已验收劳动收益不得被客户退款自动冲减或追回。
- 业绩和积分从订单冻结快照自动产生；净收款不足整单回滚，补足后恢复。

### 案例

`内部候选 → 待审核 → 已发布 → 已下架/已归档`。

- 成交和履约完成只生成内部候选，不自动公开。
- 有效客户展示授权和管理员审核缺一不可。
- 授权撤回或下架时，公开投影立即不可见并删除公开图片副本。

## 守恒与幂等不变量

- 订单行门店分配数量之和等于订单行数量。
- 预留 + 已出库不得超过订单冻结硬件数量。
- 金额事件使用追加和反向记录，不通过改写历史达到“对账”。
- 销售利润与劳动收益是两套独立事实，不能互相覆盖。
- 工作台、推进中心、日历共用同一工作项来源。
- 公开案例永远不直接查询内部案例表。
