## 创建测试环境前的用户决策

创建 Supabase 项目或数据库分支可能产生费用。必须先由项目所有者确认：

1. 使用哪个 Supabase 组织；
2. 采用独立项目还是开发分支；
3. 区域和预算；
4. 明确该项目只承载脱敏/合成测试数据。

监理取得以上确认后，先查询实际费用并再次展示给项目所有者确认，再创建资源。未确认前不创建、不收费。

## 环境隔离合同

- 生产和测试必须有不同项目 ref、URL、发布环境和变量来源。
- 预览构建只接受测试项目 ref；生产构建只接受生产项目 ref。
- 构建开始时校验 ref，发现错配立即失败。
- 浏览器只使用 publishable/anon key；service role、数据库密码和管理凭证不进入前端、仓库、文档或构建日志。
- 测试数据只使用合成或批准脱敏数据，不从测试回写生产。
- 生产 Storage 策略在 3.0 施工期保持不动；图片禁令先在测试环境验证。

## 备份包组成

1. 数据库结构和数据逻辑导出，并记录命令版本、时间、文件校验和。
2. Auth 用户标识与业务身份/岗位关系映射；不导出或记录明文密码。
3. Storage 桶、对象路径、大小、内容类型和校验和清单；文件对象必须单独备份。
4. 两个允许图片位的私有/公开对象清单。
5. Edge Functions 源码版本、部署配置和函数清单。
6. Cron/工作器计划、时区、游标和补跑配置。
7. 功能开关、环境变量名称、Git 提交号和前端产物校验和；密钥只登记托管位置。
8. 表行数、关键金额、库存、积分和收益对账清单。
