## 当前结论

P0 已启动，只做现状冻结、安全基线、测试环境和后续接口冻结。P0 未通过前，不开发 P3–P6 业务功能，不修改生产数据库和生产 Storage 策略。

当前已核实：

- 生产 Supabase 项目仍为唯一可见项目，Postgres 17.6.1；尚无独立测试项目证据。
- 生产库有 103 张 `public` 表、162 个 `public` 函数/过程、229 条 `public` 策略、248 个 `public` 索引、11 个 `public` 视图。
- 触发器按对象/名称计为 29 个，按 `information_schema.triggers` 事件行计为 46 条；总方案的 46 已确认是事件展开口径。
- 线上和本地迁移版本及名称精确匹配 69/69；本地 69 项版本/文件名/SHA256 门禁已建立，尚未证明远端已执行 SQL 正文与当前文件逐字一致，CI 也未接入。
- 安全顾问当前有 3 个 ERROR、137 个 WARN、3 个 INFO。三个 ERROR 均为安全定义者视图：`finance_public_summary`、`inventory_public_items`、`assets_public`。
- 性能顾问当前有 100 个 WARN、215 个 INFO；包含 205 个未索引外键提示、54 个 RLS 初始化计划提示、46 个重复宽松策略提示、10 个未使用索引提示。
- 当前发布工作流只安装、构建、发布，没有实际运行数据库、权限和业务测试。

以上均为只读证据，不代表修复完成，也不代表 G0 通过。

## 停止线

- 不在生产项目执行 DDL、策略、Storage、迁移、恢复或发布操作。
- 不改写现有 69 个迁移，不删除表，不改表名，不批量重写数据。
- 不把页面完成、静态检查、候选库通过、现网兼容、老板验收和允许合并混为一个状态。
- 第一次正式迁移、部署、恢复或发布失败后立即停止，保留证据，等待新决定。
