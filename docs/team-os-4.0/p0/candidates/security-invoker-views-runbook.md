# 三个 Security Invoker 视图隔离验证手册

> 候选：`security-invoker-views.sql`
> 状态：仅本地评审，禁止在生产执行。
> 一次失败即停：首次正式执行、迁移、API、页面、Advisor 或回退验证出现失败，立即保存证据并结束本次授权，不就地补 SQL、补策略或重试。

## 1. 进入条件

只有全部满足才允许开始一次隔离验证：

1. 使用全新的、可销毁且与生产隔离的 Supabase 项目；项目 ID、连接串和密钥不写仓库。
2. 数据库迁移链、Auth 身份/岗位关系和 Storage 清单已按 `06-test-project-and-restore-baseline.md` 恢复并核对；没有恢复证据不得执行候选。
3. 已保存执行前数据库快照、三视图定义/ACL/owner、底表 RLS/策略、Security Advisor 和前端构建提交号。
4. 静态闸门已通过：

   ```powershell
   powershell.exe -NoProfile -ExecutionPolicy Bypass -File scripts/p0/validate-security-invoker-view-candidate.ps1
   ```

5. 建立六个专用测试身份，不复用真实用户：未登录、销售、实施、运维、财务、管理员。每个登录身份只有一个主岗位且属于同一测试团队；另建第二测试团队数据用于跨团队负向验证。
6. 仓库和主管是附加职能，只能作为六身份基线通过后的扩展用例，不能冒充第六主岗位。

## 2. 执行前基线

对每个视图保存：

- 精确列名、顺序和类型；owner、ACL、`security_invoker` 属性；
- 同团队及第二团队底表的最小脱敏夹具行数；
- 六身份对视图和底表的直接 API 状态码/行数/字段；
- 三个前端服务调用的构建与页面结果；
- Security Advisor 分类和总数。

若基线与风险登记册不一致，停止，不执行候选。

## 3. 唯一正式执行

由被授权操作者在 Supabase SQL Editor 或已批准的迁移通道中，完整执行 `security-invoker-views.sql` 一次。执行前再次确认目标 project ref 是隔离项目，不允许复制其中任何语句到生产。

首次执行返回任意 SQLSTATE、对象/列契约错误或权限错误时：

1. 不修改候选；
2. 不第二次执行；
3. 保存原始 SQLSTATE、完整错误、时间、project ref、执行提交号和当前 Advisor；
4. 将隔离项目保留给复盘，等待新的明确授权。

## 4. 六身份直接 API 验证

每个身份必须分别对三个视图发起直接 REST/API 查询，不能只看前端按钮。至少检查同团队、第二团队和明确敏感字段。

| 身份 | `finance_public_summary` | `inventory_public_items` | `assets_public` | 必须拒绝/不可见 |
| --- | --- | --- | --- | --- |
| 未登录/anon | 无访问 | 无访问 | 无访问 | 三个视图全部拒绝，不能返回空数据伪装成功 |
| 销售 | 拒绝 | 仅允许公开库存字段和同团队行 | 仅允许公开资产字段和同团队行 | 财务汇总、内部库存字段、跨团队行 |
| 实施 | 拒绝 | 同团队公开库存；仓库动作不在本视图 | 同团队公开资产 | 财务汇总、跨团队行、写操作 |
| 运维 | 拒绝 | 同团队公开库存 | 同团队公开资产 | 财务汇总、跨团队行、写操作 |
| 财务 | 同团队财务汇总 | 同团队公开库存 | 同团队公开资产 | 跨团队行、视图写操作 |
| 管理员 | 同团队三视图 | 同团队三视图 | 同团队三视图 | 跨团队行（除非另有明确跨团队治理授权）、视图写操作 |

关键判定：`security_invoker = true` 暴露的是底表真实权限。如果任何“应允许”身份失败，或任何“应拒绝”身份成功，都属于安全/业务语义失败。立即执行一次失败即停，不临时添加宽泛 RLS、GRANT 或 SECURITY DEFINER RPC。

## 5. 契约与顾问复核

六身份全部通过后，依次核验：

1. 视图列名、顺序和类型与执行前完全一致。
2. `src/services/finance.ts`、`src/services/inventory.ts`、`src/services/assets.ts` 三个调用仍能按原字段读取；页面无 401/403/5xx 或静默空数据。
3. `PUBLIC`、`anon` 没有 ACL；`authenticated` 只有视图 `SELECT`，未由本候选获得底表新权限。
4. Security Advisor 的三个 `security_definer_view` ERROR 归零，且没有新增 ERROR/WARN。
5. 135 个 authenticated 可执行 SECURITY DEFINER、3 个 RLS 无策略、1 个可变 `search_path` 仍按独立风险处理；本候选不得声称已解决它们。

任一项首次失败，保存响应、页面截图、网络日志、Advisor 导出和数据库对象定义后停止。

## 6. 回退演练与交付证据

在同一隔离项目使用执行前快照或已评审的反向迁移完成一次回退演练。不得为回退直接改写 69 个历史迁移。回退失败同样立即停止。

一次完整候选交付包必须包含：

- 隔离项目标识、执行者、时间、提交号和候选 SHA-256；
- 恢复核对表与执行前快照位置；
- 六身份 × 三视图 × 同团队/跨团队的允许/拒绝结果；
- 三个前端调用的页面、网络和日志证据；
- 执行前后 Advisor、视图定义/ACL/owner；
- 回退结果；
- 首次失败（如有）的原始证据和“未重试”声明。

这些证据只构成生产候选评审材料，不构成生产执行授权。
