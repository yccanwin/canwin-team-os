# CanWin Team OS 3.0 当前验证报告

- 报告日期：2026-07-13
- 范围：销售优先版本地静态 80% 基线
- 结论：静态审查与本地生产构建通过；线上运行时尚未验证

## 已通过证据

1. `npm run build` 已完成 TypeScript project build 与 Vite production build，退出码为 0。
2. 最近一次构建转换 1958 个模块，并生成销售、报价、订单履约、权限和管理看板独立路由 chunk。
3. GitHub Pages base 为 `/canwin-team-os/`，应用使用 HashRouter。
4. 全仓 30 个 Supabase 迁移文件版本号唯一。
5. 3.0 五个真实路由均受默认关闭的 `sales_os_v3` 功能开关保护，不回退演示数据。
6. 销售主链已完成静态逐段复核：今日 → 建档/领取 → 跟进 → 资格/商机 → A 类演示 → 报价 → 定金/订单 → 内部款 → 精确库存 → 安装/培训 → 售后交接。
7. 关键写操作使用安全 RPC；报价和订单表的关键客户端直写通道已撤销；内部价、库存锁和履约门禁在服务端计算或校验。
8. 3.0 库存预留/出库入口仅存在于真实 `/orders-v3` 订单上下文；报价阶段不占库存。
9. Supabase 浏览器配置已改为构建环境变量；源码未保留项目 URL、anon 值、service-role 或企业微信密钥。
